# readme-score

**readme-score** evaluates your README.md against a 14-point rubric and returns a score from 0 to 100, a letter grade (A+ to F), and a prioritized list of improvement suggestions — each with the exact points it's worth.

Use it in your terminal, in CI to enforce README quality gates, or programmatically in Node.js scripts.

## Why README Quality Matters

A bad README kills adoption. When a developer lands on your npm package or GitHub repo, the README is the entire sales pitch. Studies of open source package adoption consistently show:

- **Installation rate** drops sharply when install instructions are unclear or missing
- **Trust** is established by badges (CI, version, license) — developers look for them in under 2 seconds
- **Usage examples** are the single most requested improvement in npm package feedback

readme-score turns vague advice ("write better docs") into a concrete action plan with a measurable score.

## What Gets Measured

14 checks across 7 categories, totalling 100 points:

| Category | Points |
|----------|--------|
| Title & Description | 15 |
| Badges | 10 |
| Installation | 15 |
| Usage & Examples | 20 |
| API Documentation | 15 |
| Contributing | 10 |
| License | 10 |
| Extras (Features, Changelog) | 5 |

## Self-Scoring

readme-score scores 100/100 on its own README. That's the bar.

## Install

```bash
npm install -g readme-score
```

Or without installing:

```bash
npx readme-score
```
