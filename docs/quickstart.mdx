## Install

```bash
npm install -g readme-score
```

## Score Your README

From inside any project directory:

```bash
readme-score
```

This scores `./README.md` and shows a full breakdown.

## Score a Specific File

```bash
readme-score ./packages/my-pkg/README.md
```

## See Your Score As JSON

```bash
readme-score --json
```

```json
{
  "total": 75,
  "max": 100,
  "grade": "B",
  "suggestions": [
    "Add has an api, reference, or cli reference section (+15 pts)"
  ],
  "categories": {
    "Title & Description": {
      "points_earned": 15,
      "points_available": 15,
      "checks": [...]
    }
  }
}
```

## Enforce a Minimum Score in CI

Fail the build if README quality drops below 70:

```bash
readme-score --min 70
```

In GitHub Actions:

```yaml
- name: Check README quality
  run: npx readme-score --min 70
```

## Use Programmatically

```js
const { score } = require('readme-score');
const fs = require('fs');

const content = fs.readFileSync('./README.md', 'utf8');
const result = score(content);

console.log(`Score: ${result.total}/100 (${result.grade})`);
console.log('Suggestions:', result.suggestions);
```

## Interpret Your Score

| Grade | Score | What it means |
|-------|-------|---------------|
| A+ | 90–100 | Excellent. Comprehensive and well-documented. |
| A  | 80–89  | Great. Minor tweaks to reach perfect. |
| B  | 70–79  | Good. A few sections need attention. |
| C  | 60–69  | Decent start. Several sections missing. |
| D  | 50–59  | Needs significant work. |
| F  | 0–49   | Major sections missing. |
